# 筆記編輯 Rules

	創造有組織，容易尋找還原的第二大腦

## Git規則

- Obsidian設定與 git 規則的 commit 與筆記分開
  理由：避免設定檔誤改

## 其他筆記互動規則

- 線上編輯分享使用 google doc、Notion
- google keep 快速記下
- GitHub 直接線上編輯

## Obsidian設定規則

- 快速鍵盡量跟 VSCode 一致
- 複製檔案路徑快速鍵為相對路徑
- 系統完整路徑使用Windows檔案總管

## 檔名規則

- 盡量不要有空白鍵，有空白要以 % 符號替代
- AI研究以 `Chat-` 開頭

## 文件編輯規則

- GitHub Flavor Markdown 格式相容
- 一則筆記沒有結構並且等待整理，以 `雜記-` 開頭
- 使用 AI 聊天解答以 `Chat－` 開頭（內文與文件）
